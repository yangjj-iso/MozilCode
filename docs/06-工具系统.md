# 06 - 工具系统

## 1. 概述

工具（Tool）是 Agent 与外部世界交互的手段。LLM 通过工具调用协议声明要执行的工具及参数，由宿主程序执行工具并将结果喂回 LLM。本项目的工具系统支持文件操作、命令执行、搜索、子 Agent 调用等多种工具。

对应代码：`mozilcode/tools/`

## 2. 基础概念

### 2.1 什么是工具？

在 AI Agent 的语境下，工具是一个**可执行的函数**，LLM 可以通过结构化的方式调用它。每个工具有：
- **名称**：LLM 用这个名字来调用工具
- **描述**：告诉 LLM 这个工具做什么
- **参数 Schema**：定义工具接受什么参数（JSON Schema 格式）
- **执行函数**：实际干活的代码

```
LLM: "我需要读取 main.py 文件"
     → tool_call(name="ReadFile", arguments={"path": "main.py"})

宿主程序:
     → 执行 ReadFile 工具
     → 返回 ToolResult(output="文件内容...", is_error=False)
```

### 2.2 JSON Schema

JSON Schema 是描述 JSON 数据结构的标准。在本项目中，每个工具用 JSON Schema 描述它接受的参数：

```json
{
  "name": "ReadFile",
  "description": "读取文件内容",
  "input_schema": {
    "type": "object",
    "properties": {
      "path": {
        "type": "string",
        "description": "文件路径"
      },
      "offset": {
        "type": "integer",
        "description": "起始行号",
        "default": 0
      }
    },
    "required": ["path"]
  }
}
```

### 2.3 抽象基类（ABC）

Python 的 `ABC`（Abstract Base Class）定义了子类必须实现的方法：

```python
from abc import ABC, abstractmethod

class Tool(ABC):
    name: str
    description: str
    params_model: type[BaseModel]  # Pydantic 模型

    @abstractmethod
    async def execute(self, params: BaseModel) -> ToolResult:
        ...  # 子类必须实现
```

## 3. Tool 基类

```python
@dataclass
class ToolResult:
    output: str           # 工具输出（文本）
    is_error: bool = False  # 是否出错

class Tool(ABC):
    name: str                          # 工具名称
    description: str                   # 工具描述
    params_model: type[BaseModel]      # 参数 Pydantic 模型
    category: ToolCategory = "read"   # 类别: read | write | command
    is_concurrency_safe: bool = False  # 是否可以并发执行
    is_system_tool: bool = False       # 是否系统工具
    should_defer: bool = False         # 是否延迟注册（按需发现）

    @property
    def is_read_only(self) -> bool:
        return self.category == "read"

    def get_schema(self) -> dict[str, Any]:
        """生成 JSON Schema，发给 LLM。"""
        schema = self.params_model.model_json_schema()
        return {
            "name": self.name,
            "description": self.description,
            "input_schema": schema,
        }

    @abstractmethod
    async def execute(self, params: BaseModel) -> ToolResult:
        """执行工具，返回结果。"""
```

### ReadFile 示例

```python
class ReadFileParams(BaseModel):
    path: str
    offset: int = 0
    limit: int = 0

class ReadFile(Tool):
    name = "ReadFile"
    description = "读取文件内容"
    params_model = ReadFileParams
    category = "read"
    is_concurrency_safe = True

    def __init__(self, file_cache=None, file_state_cache=None, base_dir=None):
        self._cache = file_cache
        self._state_cache = file_state_cache
        self._base_dir = base_dir

    async def execute(self, params: ReadFileParams) -> ToolResult:
        path = self._resolve_path(params.path)
        content = path.read_text(encoding="utf-8")
        return ToolResult(output=content)
```

## 5. ToolRegistry（工具注册表）

```python
class ToolRegistry:
    def __init__(self) -> None:
        self._tools: dict[str, Tool] = {}      # 名称 → 工具
        self._disabled: set[str] = set()        # 已禁用的工具
        self._discovered: set[str] = set()      # 已发现的延迟工具

    def register(self, tool: Tool) -> None:
        if tool.name in self._tools:
            raise ToolRegistryError(f"tool already registered: {tool.name}")
        self._tools[tool.name] = tool

    def get(self, name: str) -> Tool | None:
        return self._tools.get(name)

    def list_tools(self) -> list[Tool]:
        return list(self._tools.values())

    def get_all_schemas(self, protocol: str) -> list[dict]:
        """获取所有已启用工具的 Schema。"""
        return [
            schema_for_protocol(tool, protocol)
            for name, tool in self._tools.items()
            if name not in self._disabled
        ]
```

### 延迟注册（Deferred Tools）

有些工具（如 MCP 工具）数量可能很多，全发给 LLM 会浪费 token。`should_defer` 标记的工具不会出现在初始工具列表中，而是通过 `ToolSearch` 工具按需发现：

```python
def get_deferred_tool_names(self) -> list[str]:
    """返回尚未发现的延迟工具名称。"""
    return [
        name for name, tool in self._tools.items()
        if self._is_deferred(tool)
        and name not in self._discovered
        and name not in self._disabled
    ]
```

## 6. 工具创建工厂

```python
def create_default_registry(
    file_cache=None, file_history=None, base_dir=None
) -> ToolRegistry:
    registry = ToolRegistry()

    # 只读工具
    registry.register(ReadFile(file_cache=file_cache, base_dir=base_dir, ...))
    registry.register(Glob(base_dir=base_dir))
    registry.register(Grep(base_dir=base_dir))

    # 写入工具
    registry.register(WriteFile(file_cache=file_cache, file_history=file_history, ...))
    registry.register(EditFile(file_cache=file_cache, file_history=file_history, ...))

    # 命令工具
    registry.register(Bash())

    return registry
```

## 7. 协议适配

不同 LLM 协议对工具 Schema 的格式要求不同：

```python
def schema_for_protocol(tool: Tool, protocol: str) -> dict[str, Any]:
    base = tool.get_schema()
    if protocol in ("openai", "openai-compat"):
        # OpenAI 格式
        return {
            "type": "function",
            "name": base["name"],
            "description": base["description"],
            "parameters": base["input_schema"],
        }
    # Anthropic 格式（直接使用 base）
    return base
```

## 8. 工具执行流程

```
LLM 返回 tool_call
       ↓
partition_tool_calls()  ← 分批：并发安全的放一批，串行的逐个
       ↓
┌─ 并发批次 ─────────────────────────────────┐
│  for tc in batch.calls:                     │
│    1. run_pre_tool_hook()  ← 前置 Hook       │
│    2. authorize_tool_call() ← 权限检查        │
│       ↓ approved                            │
│  asyncio.gather(*[execute(tc) for tc])     │
│    3. tool.execute(params) ← 并行执行        │
│    4. run_post_tool_hook() ← 后置 Hook       │
└─────────────────────────────────────────────┘

┌─ 串行批次 ─────────────────────────────────┐
│  for tc in batch.calls:                     │
│    1. run_pre_tool_hook()                   │
│    2. authorize_tool_call()                 │
│    3. tool.execute(params)  ← 单个执行       │
│    4. run_post_tool_hook()                  │
│    5. _snapshot_for_recovery()              │
└─────────────────────────────────────────────┘
       ↓
conversation.add_tool_results_message(results)
       ↓
进入下一轮 LLM 调用
```

## 9. FileStateCache

文件工具使用 `FileStateCache` 来跟踪文件状态，防止 LLM 基于过期的文件内容做决策：

```python
class FileStateCache:
    """记录文件最后被读取/修改时的状态。

    - WriteFile/EditFile 执行后，更新文件状态
    - ReadFile 执行前，检查文件是否在上次读取后被外部修改
    """
    def __init__(self):
        self._states: dict[str, float] = {}  # path → mtime

    def update(self, path: str, mtime: float):
        self._states[path] = mtime

    def is_stale(self, path: str, current_mtime: float) -> bool:
        """文件是否在上次缓存后被修改过。"""
        return self._states.get(path, 0) < current_mtime
```

## 10. Worktree 工具适配

子 Agent 在 Git Worktree 中工作时，需要将文件工具的 `base_dir` 指向 worktree 路径：

```python
def rebase_file_tools(registry: ToolRegistry, base_dir: str) -> ToolRegistry:
    """克隆文件工具，让相对路径从 base_dir 解析。"""
    rebased = ToolRegistry()
    for tool in registry.list_tools():
        if isinstance(tool, ReadFile):
            replacement = ReadFile(base_dir=base_dir, ...)
        elif isinstance(tool, WriteFile):
            replacement = WriteFile(base_dir=base_dir, ...)
        elif isinstance(tool, EditFile):
            replacement = EditFile(base_dir=base_dir, ...)
        elif isinstance(tool, Glob):
            replacement = Glob(base_dir=base_dir)
        elif isinstance(tool, Grep):
            replacement = Grep(base_dir=base_dir)
        else:
            replacement = tool  # 非文件工具直接复用
        rebased.register(replacement)
    return rebased
```

## 11. AgentTool（子 Agent 调用）

`AgentTool` 允许 LLM 启动子 Agent 来处理子任务：

```python
class AgentTool(Tool):
    name = "Agent"
    description = "启动子 Agent 执行子任务"
    category = "command"

    async def execute(self, params) -> ToolResult:
        # 1. 从 AgentLoader 获取子 Agent 定义
        agent_def = self._agent_loader.get(params.agent_name)

        # 2. 创建子 Agent 实例
        child_agent = await self._create_child_agent(agent_def)

        # 3. 在后台运行子 Agent
        task = asyncio.create_task(child_agent.run_to_completion(params.prompt))

        # 4. 返回子 Agent 的最终结果
        result = await task
        return ToolResult(output=result)
```

这使得 Agent 可以委托子任务给专门的子 Agent（如代码审查 Agent、测试 Agent 等）。

## 12. 工具系统与 Agent / Daemon 的边界（补充）

### 12.1 谁创建工具，谁执行工具

`	ext
create_agent_from_config(work_dir)
  → create_default_registry(base_dir=work_dir)
  → MCPManager.register_all_tools(registry)
  → Agent(registry=...)

Agent.run 工具阶段
  → tool_execution / authorize / hooks
  → tool.execute(params)
  → ToolResult 写回 conversation
  →（Daemon）serialize ToolUse/ToolResult 事件给 GUI
`

要点：

- **工具实例绑 work_dir**（文件类工具的 base_dir / sandbox）
- Daemon 不直接执行工具；它只跑 gent.run
- MCP 工具与内置工具对 Agent 来说都是 Tool 接口

### 12.2 ToolRegistry 的三种可见性

| API 概念 | 含义 |
|----------|------|
| 已注册 | name → Tool 在目录中 |
| 启用/禁用 | 禁用后不出现在默认 schema |
| deferred（should_defer） | 默认不进 schema，需 ToolSearch 发现后再用 |

Deferred 常见于大量 MCP 工具，避免 tools schema 爆炸。

### 12.3 并发安全与批次

主循环会把一轮 tool_calls 分成：

- **并发批次**：只读/标记为 concurrent-safe 的工具
- **串行批次**：写文件、bash、改状态类

权限与 pre_tool Hook 在执行前；post_tool Hook 在执行后。写工具后可能 snapshot 供压缩恢复。

### 12.4 子 Agent 工具链

AgentTool：

1. 解析子 Agent 定义（defaults / loader）
2. 过滤可用工具（tool_filter）
3. 可能 rebase 文件工具到 worktree
4. 非交互路径执行
5. 结果作为父 Agent 的一个 ToolResult
