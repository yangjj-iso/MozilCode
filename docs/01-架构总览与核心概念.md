# 01 - 项目架构总览与核心概念

## 1. 这是什么项目？

MozilCode 是一个 **AI 编程助手运行时**（AI Coding Assistant Runtime），类似于 Claude Code、Cursor 等 AI 编程工具的后端引擎。它的核心能力是：

1. 接收用户的自然语言指令
2. 调用大语言模型（LLM）进行推理
3. 让 LLM 通过"工具调用"（Tool Use）操作文件系统、执行命令
4. 循环上述过程直到任务完成

```
用户: "帮我创建一个 hello.py 文件"
       ↓
  Agent 主循环
       ↓
  LLM 推理 → 决定调用 WriteFile 工具
       ↓
  权限检查 → 执行 WriteFile
       ↓
  工具结果返回给 LLM → LLM 生成最终回复
       ↓
用户看到: "已创建 hello.py"
```

## 2. 基础概念扫盲

### 2.1 LLM（大语言模型）

LLM（Large Language Model）是一种基于深度学习的语言模型，通过预测"下一个词"来生成文本。常见的 LLM 有 GPT-4、Claude 等。

在本项目中，LLM 的角色是**大脑**——它理解用户的意图，决定调用什么工具，并根据工具返回的结果生成回复。

### 2.2 Token（词元）

Token 是 LLM 处理文本的基本单位。一个 Token 大致等于 3-4 个英文字符或 1-2 个中文字符。LLM 有一个**上下文窗口**（Context Window）限制，例如 200,000 tokens，意味着它一次最多能"看到"这么多内容。

```python
# 项目中的 token 估算方法（conversation.py）
_CHARS_PER_TOKEN = 3.5  # 大约每 3.5 个字符算 1 个 token

def estimate_tokens(messages: list[Message]) -> int:
    total = sum(_message_chars(m) for m in messages)
    return int(total / _CHARS_PER_TOKEN)
```

### 2.3 工具调用（Tool Use / Function Calling）

工具调用是 LLM 与外部世界交互的机制。LLM 本身只能生成文本，但通过工具调用协议，LLM 可以"声明"它想执行某个工具及参数，由宿主程序代为执行，然后将结果喂回给 LLM。

```
轮次 1:
  用户消息: "读取 config.yaml 的内容"
  → LLM 返回: tool_call(name="ReadFile", arguments={"path": "config.yaml"})

宿主程序:
  执行 ReadFile 工具 → 得到文件内容

轮次 2:
  将工具结果喂回 LLM
  → LLM 返回: "这个配置文件的内容是..."
```

### 2.4 异步编程（asyncio）

Python 的 asyncio 是一种并发编程方式，用 `async/await` 语法编写。在本项目中，几乎所有 I/O 操作（网络请求、文件读写、工具执行）都是异步的。

```python
# 异步函数定义
async def fetch_data():
    await asyncio.sleep(1)  # 模拟耗时操作
    return "data"

# 并发执行多个任务
results = await asyncio.gather(task1(), task2(), task3())
```

**为什么需要异步？** 因为 LLM 的 API 调用是网络 I/O，可能需要几秒到几十秒。在此期间，异步编程允许程序做其他事情（如处理其他请求），而不是傻等。

### 2.5 AsyncIterator（异步迭代器）

本项目大量使用异步迭代器来处理**流式响应**。LLM 的回复不是一次性返回的，而是一个字一个字地"流"出来的。

```python
# Agent.run() 就是一个异步迭代器
async def run(self, conversation) -> AsyncIterator[AgentEvent]:
    while True:
        # ... 每次循环 yield 出事件
        yield StreamText(text="LLM 回复的片段")
        yield ToolUseEvent(tool_name="ReadFile", ...)
        yield ToolResultEvent(output="文件内容...")
        yield LoopComplete(total_turns=3)

# 消费者
async for event in agent.run(conversation):
    if isinstance(event, StreamText):
        print(event.text, end="", flush=True)
```

## 5. 核心数据流

下面用伪代码描述一次完整的用户交互流程：

```python
# 1. 用户发送消息
conversation.add_user_message("帮我读取 main.py")

# 2. Agent 开始主循环
async for event in agent.run(conversation):
    # 3. Agent 调用 LLM，流式接收响应
    if isinstance(event, StreamText):
        # LLM 正在输出文本，实时显示给用户
        ui.display(event.text)

    # 4. LLM 决定调用工具
    elif isinstance(event, ToolUseEvent):
        # 例如: tool_name="ReadFile", arguments={"path": "main.py"}
        ui.show_tool_call(event.tool_name, event.arguments)

    # 5. 工具执行完成
    elif isinstance(event, ToolResultEvent):
        # 工具结果会被自动喂回 LLM，进入下一轮循环
        ui.show_tool_result(event.output)

    # 6. LLM 给出最终回复，循环结束
    elif isinstance(event, LoopComplete):
        ui.display("任务完成")
        break
```

## 6. 关键设计决策

### 6.1 Headless 架构

Agent 核心是**完全无界面**的（headless），只负责接收对话、调用 LLM、执行工具、输出事件流。所有界面（TUI、CLI、Web GUI）都是事件流的消费者。

```python
# Agent 不关心谁在消费它的事件
async def run(self, conversation) -> AsyncIterator[AgentEvent]:
    # ...
    yield StreamText(text="...")
    yield ToolUseEvent(...)
    yield LoopComplete(...)
```

### 6.2 事件驱动

Agent 与外界的所有交互都通过**事件**完成。事件用 dataclass 定义，清晰且类型安全。

```python
@dataclass
class StreamText:        # LLM 输出的文本片段
    text: str

@dataclass
class ToolUseEvent:      # LLM 请求调用工具
    tool_name: str
    tool_id: str
    arguments: dict[str, Any]

@dataclass
class PermissionRequest:  # 需要用户授权
    tool_name: str
    description: str
    future: asyncio.Future[PermissionResponse]  # 异步等待用户回复

@dataclass
class LoopComplete:       # 主循环结束
    total_turns: int
```

### 6.3 配置驱动

所有行为（模型选择、权限模式、MCP 服务器、记忆系统等）都通过 YAML 配置文件控制，支持多层合并：

```
~/.mozilcode/config.yaml          # 用户级默认配置
.mozilcode/config.yaml            # 项目级配置
.mozilcode/config.local.yaml      # 本地覆盖（不提交到 git）
```

### 6.4 多协议支持

通过 Provider 适配层，支持三种 LLM API 协议：

| 协议 | 说明 | 典型 Provider |
|------|------|--------------|
| `anthropic` | Claude API | Anthropic 官方 |
| `openai` | OpenAI API | OpenAI 官方 |
| `openai-compat` | OpenAI 兼容接口 | 各种第三方 API |

```yaml
# config.yaml 示例
providers:
  - name: agnes
    protocol: openai-compat
    base_url: https://apihub.agnes-ai.com/v1
    model: agnes-2.0-flash
    api_key: ${AGNES_API_KEY}
```
