# 04 - 配置系统

## 1. 概述

配置系统管理整个项目的运行参数：模型 Provider、权限模式、MCP 服务器、记忆系统、Hook、技能等。采用 YAML 格式，支持多层合并。

对应代码：`mozilcode/config/`

## 2. 基础概念

### 2.1 YAML 格式

YAML（YAML Ain't Markup Language）是一种人类友好的数据序列化格式，广泛用于配置文件。

```yaml
# YAML 基本语法
providers:                    # 列表
  - name: openai              # 列表项是字典
    protocol: openai
    base_url: https://api.openai.com/v1
    model: gpt-4.1
    api_key: ${OPENAI_API_KEY}  # 环境变量引用
    thinking: false             # 布尔值
    context_window: 0           # 整数

permission_mode: default        # 字符串

mcp_servers:                    # 另一个列表
  - name: filesystem
    command: npx
    args: ["-y", "@modelcontextprotocol/server-filesystem"]
```

### 2.2 Dataclass（数据类）

Python 的 `dataclass` 装饰器自动生成 `__init__`、`__repr__` 等方法，非常适合表示配置结构：

```python
from dataclasses import dataclass, field

@dataclass
class ProviderConfig:
    name: str
    protocol: str
    base_url: str
    model: str
    api_key: str = ""           # 有默认值的字段放后面
    thinking: bool = False
    context_window: int = 0
    max_output_tokens: int = 0
```

### 2.3 环境变量替换

配置文件中可以用 `${VAR_NAME}` 语法引用环境变量，避免在配置文件中硬编码密钥：

```python
import re

_ENV_VAR_RE = re.compile(r"\$\{([^}]+)\}")

def resolve_env_vars(value: str) -> str:
    """将 ${VAR_NAME} 替换为对应的环境变量值。"""
    return _ENV_VAR_RE.sub(
        lambda m: os.environ.get(m.group(1), m.group(0)),  # 找不到则保留原文
        value
    )
```

## 3. 配置文件层级

配置从三个位置加载，按顺序合并：

```
1. ~/.mozilcode/config.yaml          ← 用户级默认配置
2. .mozilcode/config.yaml            ← 项目级配置（覆盖用户级）
3. .mozilcode/config.local.yaml      ← 本地覆盖（不提交到 git）
```

```python
def load_config(path: Path | None = None) -> AppConfig:
    if path is not None:
        return _load_single_file(path)  # 指定路径，不合并

    candidates = [
        home / ".mozilcode" / "config.yaml",
        cwd / ".mozilcode" / "config.yaml",
        cwd / ".mozilcode" / "config.local.yaml",
    ]

    merged = None
    for p in candidates:
        if not p.exists():
            continue
        layer = _load_single_file(p, require_providers=False)
        if merged is None:
            merged = layer
        else:
            merged = _merge_config(merged, layer)

    if merged is None:
        raise ConfigError("No config file found")
    if not merged.providers:
        raise ConfigError("At least one provider must be configured")
    return merged
```

## 4. 配置结构

### 4.1 AppConfig 顶层结构

```python
@dataclass
class AppConfig:
    providers: list[ProviderConfig]          # LLM Provider 列表（至少 1 个）
    permission_mode: str = "default"          # 权限模式
    mcp_servers: list[MCPServerConfig]        # MCP 服务器列表
    raw_hooks: list[dict]                     # Hook 配置
    memory: MemoryConfig                      # 记忆系统配置
    worktree: WorktreeConfig                  # Git Worktree 配置
    teammate_mode: str = ""                   # 多智能体模式
    enable_fork: bool = False                 # 是否启用 fork
    enable_verification_agent: bool = False   # 是否启用验证 Agent
    enable_coordinator_mode: bool = False     # 是否启用协调器模式
```

### 4.2 ProviderConfig

```python
@dataclass
class ProviderConfig:
    name: str               # 唯一名称，如 "openai"
    protocol: str           # "anthropic" | "openai" | "openai-compat"
    base_url: str           # API 基础 URL
    model: str              # 模型名称，如 "gpt-4.1"
    api_key: str = ""       # API 密钥
    thinking: bool = False  # 是否启用 thinking 模式
    context_window: int = 0     # 上下文窗口（0 = 自动解析）
    max_output_tokens: int = 0  # 最大输出 token（0 = 自动）
```

### 4.3 MCPServerConfig

```python
@dataclass
class MCPServerConfig:
    name: str
    command: str | None = None  # stdio 模式：启动命令
    args: list[str] = []        # 命令参数
    url: str | None = None      # HTTP 模式：服务器 URL
    headers: dict[str, str] = {}  # HTTP 头
    env: dict[str, str] = {}      # 环境变量

    @property
    def is_stdio(self) -> bool:
        return self.command is not None
```

## 5. 配置校验

校验逻辑在 `validator.py` 中，采用**严格校验 + 清晰错误信息**的策略：

```python
def validate_providers(raw_providers: list) -> list[dict]:
    if not isinstance(raw_providers, list) or len(raw_providers) == 0:
        raise ConfigError("At least one provider must be configured")

    for i, entry in enumerate(raw_providers):
        # 检查必填字段
        missing = [f for f in ("name", "protocol", "base_url", "model") if f not in entry]
        if missing:
            raise ConfigError(f"Provider #{i+1}: missing fields: {', '.join(missing)}")

        # 检查 protocol 取值
        protocol = entry["protocol"]
        if protocol not in VALID_PROTOCOLS:
            raise ConfigError(
                f"Provider #{i+1}: invalid protocol '{protocol}', "
                f"must be one of: {', '.join(sorted(VALID_PROTOCOLS))}"
            )

        # 检查名称唯一性
        if name in seen_names:
            raise ConfigError(f"Provider '{name}': duplicate name")
```

### 校验流程

```
config.yaml
    ↓
yaml.safe_load()        # YAML 解析
    ↓
validate_config_structure()  # 结构校验
    ├── reject_removed_config_sections()  # 拒绝已移除的功能段
    ├── validate_providers()              # Provider 校验
    ├── validate_permission_mode()        # 权限模式校验
    ├── validate_mcp_servers()            # MCP 服务器校验
    ├── validate_memory()                 # 记忆配置校验
    └── validate_worktree()               # Worktree 配置校验
    ↓
AppConfig（数据类实例）
```

## 6. 配置合并策略

当多个配置文件存在时，合并策略是"后者覆盖前者"：

```python
def _merge_config(base: AppConfig, override: AppConfig) -> AppConfig:
    # Provider：override 完全替换
    if override.providers:
        base.providers = override.providers

    # Permission mode：只在 override 显式声明时才覆盖
    if override.permission_mode_declared:
        base.permission_mode = override.permission_mode

    # MCP servers：按名称合并（同名替换，不同名追加）
    if override.mcp_servers:
        by_name = {s.name: i for i, s in enumerate(base.mcp_servers)}
        for s in override.mcp_servers:
            if s.name in by_name:
                base.mcp_servers[by_name[s.name]] = s  # 替换
            else:
                base.mcp_servers.append(s)              # 追加

    # Hooks：追加（不替换）
    base.raw_hooks.extend(override.raw_hooks)

    # Memory：override 整体替换
    if override.memory_declared:
        base.memory = override.memory

    return base
```

## 7. Removed Capabilities（功能移除守卫）

项目有意不支持某些功能（如云端管理、QQ Bot 等）。`removed_capabilities.py` 确保配置文件和路由中不会意外引入这些功能：

```python
REMOVED_CONFIG_SECTIONS = frozenset({
    "accounts", "auth", "bot_adapters", "bots", "cloud",
    "frontend", "gui", "hosted_models", "login", "official_models",
    "qqbot", "telegrambot",
})

def find_removed_config_sections(raw: Mapping) -> tuple[str, ...]:
    """返回配置文件中不应存在的顶级段名。"""
    return tuple(sorted(
        k for k in raw if isinstance(k, str) and k in REMOVED_CONFIG_SECTIONS
    ))

def assert_no_removed_route_paths(paths: list[str]) -> None:
    """确保路由中不包含已移除的产品接口路径。"""
    removed = find_removed_route_paths(paths)
    if removed:
        raise RuntimeError(f"Removed route(s) not supported: {', '.join(removed)}")
```

## 8. 完整配置示例

```yaml
# ~/.mozilcode/config.yaml

# LLM Provider（必填，至少 1 个）
providers:
  - name: agnes
    protocol: openai-compat
    base_url: https://apihub.agnes-ai.com/v1
    model: agnes-2.0-flash
    api_key: ${AGNES_API_KEY}
    thinking: false
    context_window: 0           # 0 = 自动解析
    max_output_tokens: 0        # 0 = 自动

# 权限模式
permission_mode: default         # default | acceptEdits | plan | bypassPermissions

# MCP 服务器
mcp_servers:
  - name: filesystem
    command: npx
    args: ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"]

# 记忆系统
memory:
  enabled: true
  providers:
    - name: markdown
      type: builtin.markdown
      enabled: true

# Hook 钩子
hooks:
  - id: auto-format
    event: post_tool_use
    condition:
      tool_name: WriteFile
    action:
      type: command
      command: "prettier --write {{file_path}}"

# Worktree 配置
worktree:
  symlink_directories: ["node_modules", ".venv", "vendor"]
  stale_cleanup_interval: 3600
  stale_cutoff_hours: 24

# 多智能体
teammate_mode: ""               # "" | "in-process"
enable_fork: false
enable_verification_agent: false
enable_coordinator_mode: false
```

## 9. 配置如何进入运行时（补充）

### 9.1 加载链路

`	ext
load_config()
  读用户级 / 项目级 / local YAML
  环境变量  替换
  合并
  校验 → AppConfig
       │
       ├─ TUI：启动时加载，构造 Agent
       ├─ Daemon：run_daemon → create_app(config, work_dir)
       │     每个 session 可再按 provider_name 重排 providers
       └─ GUI：GET/POST /api/config 读写（注意 applies_to=new_sessions）
`

### 9.2 会话级 vs 全局级

| 项 | 作用域 | 说明 |
|----|--------|------|
| providers / model | 全局配置；会话创建时可选 provider_name | 已存在 runtime 不自动重建 |
| permission_mode | 默认来自配置；会话可 POST .../mode 切换 | 影响 PermissionChecker |
| mcp_servers | 创建 Agent 时 MCPManager.load_configs | 改配置后旧会话未必重连 |
| memory | 创建 Agent 时 build_memory_hub | 与 work_dir 相关 |
| hooks | Daemon 启动 load_hooks | 引擎级共享 |
| work_dir | **会话 meta**，不是 config.yaml 主字段 | 见 Daemon 文档 |

### 9.3 配置校验失败时 Daemon 行为


un_daemon：

- load_config 失败 → warning，config=None 仍可启动（健康检查等）
- 但创建需要模型的会话会因 “model provider is not configured” 失败

### 9.4 Removed Capabilities

config/removed_capabilities.py 与路由 ssert_no_removed_route_paths 配合：

- 防止配置或路由重新引入已删除功能名/路径
- 升级文档/配置模板时要同步该守卫列表

### 9.5 推荐的本地配置习惯

`	ext
~/.mozilcode/config.yaml          # 个人默认 provider / 权限习惯
项目/.mozilcode/config.yaml       # 团队共享（可提交）
项目/.mozilcode/config.local.yaml # 本机密钥与实验项（勿提交）
`

密钥优先 ${ENV}，不要把真实 key 写进可提交 YAML。
