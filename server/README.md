# server

`server` 是 Java 云端控制面预留目录。它不是本地 Agent 的替代品，而是负责用户、设备、任务和策略治理。

## 职责

- 用户登录和组织管理。
- 套餐、配额和计费。
- 模型网关和模型路由。
- 设备绑定和本地 Runtime 注册。
- 任务线程列表和状态同步。
- 手机端和本地 Runtime 之间的安全中继。
- 工具审批队列。
- 审计日志。
- 策略下发，例如工具白名单、命令黑名单、模型限制。

## 手机控制任务链路

```text
Mobile Web / App
-> Java Server
-> Secure Relay
-> Local Runtime
-> AgentLoop
-> Tools
```

Java 云端不应该直接读写用户本地文件，也不应该直接执行 Shell。它负责认证、授权、同步、审批和审计。

## 不默认保存什么

- 完整代码文件。
- 本地 Shell 输出全文。
- 用户密钥。
- 未经脱敏的敏感上下文。

如需上传上下文，必须经过策略控制和用户或企业授权。
